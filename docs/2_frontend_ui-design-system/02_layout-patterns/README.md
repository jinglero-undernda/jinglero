# Layout Patterns

## Purpose

This section documents page-level layout patterns, route mappings, grid systems, and responsive behavior. Layout patterns define how components are arranged on pages.

## Structure

### Route Mapping

See `routes.md` for complete technical mapping of all application routes to components and page templates.

### Grid System

See `grid-system.md` for grid layout, breakpoints, and container patterns.

### Responsive Behavior

See `responsive-behavior.md` for breakpoint behavior and responsive design patterns.

### Page Templates

Page templates define the layout structure for different page types:

- **Landing Page** (`page-templates/landing-page.md`) - Home page layout
- **Detail Page** (`page-templates/detail-page.md`) - Entity inspection pages
- **Admin Page** (`page-templates/admin-page.md`) - Admin interface layout
- **Search Results Page** (`page-templates/search-results-page.md`) - Search results layout

## Usage

Layout patterns use design tokens from `../01_system-foundation/tokens/` and are implemented by components documented in `../03_components/`.
