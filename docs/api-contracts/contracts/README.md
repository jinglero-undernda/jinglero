# API Contracts

## Purpose

This directory contains detailed API contract documentation for each API (Public API, Admin API, etc.). Each contract document specifies endpoints, request/response formats, validation rules, and versioning information.

## Contract Documents

_Contract documents will be created when documenting APIs using PLAYBOOK_01._

## Contract Structure

Each contract document should include:

1. **Metadata**
   - API name and version
   - Status (draft | current_implementation | validated | implemented | deprecated)
   - Last updated date

2. **Endpoint Specifications**
   - Endpoint paths and methods
   - Request parameters
   - Request body formats
   - Response formats
   - Error responses

3. **Validation Rules**
   - Parameter validation
   - Data validation
   - Constraint validation

4. **Code References**
   - Route handler locations
   - Validation code locations
   - Example usage
