# Rename Playbooks with Number Prefixes and Add Date Confirmation

## Overview

This plan will:

1. Rename all 64 playbook files to include area and playbook number prefixes
2. Update all cross-references in README files, QUICK_START files, and other documentation
3. Add date confirmation instruction to all playbooks

## Numbering Scheme

Format: `PLAYBOOK_XX_YY_DESCRIPTION.md`

- `XX` = Area number (01-09, or 00 for orchestrator)
- `YY` = Playbook number within area (01-07)

### Area Mapping

- `01`: 1_frontend_ux-workflows
- `02`: 2_frontend_ui-design-system
- `03`: 3_system_architecture
- `04`: 4_backend_database-schema
- `05`: 5_backend_api-contracts
- `06`: 6_system_testing
- `07`: 7_ops_performance
- `08`: 8_ops_security
- `09`: 9_ops_deployment
- `00`: PLAYBOOK_00_PRODUCT_ORCHESTRATOR (stays as 00, no area prefix needed)

### Example Renames

- `docs/1_frontend_ux-workflows/playbooks/PLAYBOOK_01_DOCUMENT_WORKFLOW.md` → `PLAYBOOK_01_01_DOCUMENT_WORKFLOW.md`
- `docs/1_frontend_ux-workflows/playbooks/PLAYBOOK_05_IMPLEMENT_REFACTOR.md` → `PLAYBOOK_01_05_IMPLEMENT_REFACTOR.md`
- `docs/2_frontend_ui-design-system/playbooks/PLAYBOOK_01_DOCUMENT_DESIGN_INTENT.md` → `PLAYBOOK_02_01_DOCUMENT_DESIGN_INTENT.md`

## Implementation Steps

### Step 1: Create Rename Mapping

Create a complete mapping of all 64 playbooks with their new names:

- 7 playbooks × 9 areas = 63 area playbooks
- 1 orchestrator playbook = 1 special playbook
- Total: 64 playbooks

### Step 2: Rename All Playbook Files

Rename all playbook files using the mapping. Files to rename:

- All files matching `PLAYBOOK_*.md` in each area's `playbooks/` directory
- `docs/PLAYBOOK_00_PRODUCT_ORCHESTRATOR.md` (stays as is, already has 00)

### Step 3: Update Cross-References

Update all references to playbooks in:

- `docs/README.md` - Main documentation index
- All `playbooks/README.md` files (9 files)
- All `playbooks/QUICK_START.md` files (9 files)
- Playbook files themselves (when they reference other playbooks)
- `docs/orchestrator/*.md` files
- Any validation reports or other docs that reference playbooks

### Step 4: Add Date Confirmation Instruction

Add a standardized date confirmation instruction to all playbooks. The instruction should:

- Be placed early in the "Instructions for AI Assistant" section
- Require confirming the date with the user before generating any documentation
- Apply to all date fields: Change History tables, "Last Updated" fields, workflow tables, implementation summaries

**Standard instruction to add:**

```markdown
### Step 0: Confirm Date with User

**Before generating any documentation:**
1. **Always confirm the current date with the user** before adding any date fields
2. **Ask the user**: "What is the current date? (YYYY-MM-DD format)"
3. **Use the confirmed date** for all date fields in the documentation:
   - Change History tables
   - "Last Updated" fields
   - Workflow status tables
   - Implementation summaries
   - Any other date fields

**Never assume or guess the date.**
```
